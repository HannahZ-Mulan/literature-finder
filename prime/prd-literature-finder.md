# PRD: Literature Finder - 智能文献查找助手

## 1. Introduction/Overview

Literature Finder是一个智能文献查找和管理的综合性工具，旨在帮助研究人员、学者和学生快速找到、组织、理解并引用相关的学术文献。系统集成了多个主流学术数据库，提供AI驱动的文献摘要、智能推荐和引用管理功能，让文献检索和管理更加高效便捷。

**核心价值：**
- 验证文献真实性：所有文献均来自权威学术数据库
- 智能文献总结：AI自动生成不同粒度的文献摘要
- 智能匹配推荐：基于用户论文草稿自动推荐相关文献
- 高效文献组织：分类存储、关键词标签、阅读列表
- 一站式引用管理：支持多种格式导出，直接访问DOI

## 2. Goals

- 允许用户通过关键词、作者、DOI、标题等多种方式搜索文献
- 集成多个主流学术数据库API（arXiv, PubMed, Google Scholar, Semantic Scholar等）
- 提供AI驱动的文献摘要，支持可配置的摘要长度等级
- 基于用户上传的论文草稿自动推荐相关参考文献
- 提供文献分类存储和关键词标签系统
- 支持创建和管理多个阅读列表
- 支持PDF预览和在线注释功能
- 支持多种引用格式导出（BibTeX, APA, MLA, Chicago等）
- 实现用户认证（邮箱/密码 + Google OAuth）
- 提供文献元数据展示（标题、作者、摘要、关键词、DOI、发表时间等）
- 支持直接点击DOI或链接访问原始文献

## 7. Technical Considerations

### 技术栈
- **前端：** React 18 + TypeScript + Vite
- **UI框架：** shadcn/ui + Tailwind CSS
- **状态管理：** Zustand 或 Redux Toolkit
- **后端：** Node.js + Express 或 Next.js API Routes
- **数据库：** SQLite（使用Drizzle ORM或better-sqlite3）
- **缓存：** Redis（用于API响应和摘要缓存）
- **认证：** JWT + NextAuth.js
- **AI集成：**
  - 主：智谱GLM API（便宜，中文好）
  - 备选：OpenAI GPT-4（效果最好）
  - 本地：Ollama（完全免费，需服务器）
  - 测试：Mock模式（开发测试用）
- **PDF处理：** PDF.js
- **文件存储：** 本地文件系统 或 腾讯云 COS (Cloud Object Storage)

### 已知约束
- 学术API有速率限制，需要实现请求队列和缓存
- LLM API调用有成本，需要控制使用量（详见AI成本分析）
  - GLM API: 约 ¥0.0005/次（中等摘要），成本可控
  - 建议实现缓存机制避免重复生成
  - 可通过免费配额+付费会员模式覆盖成本
- PDF处理需要考虑性能优化
- 大量文献数据需要分页加载
- SQLite单写者并发限制：需要实现写操作的排队机制
- SQLite在高并发场景下的性能限制：读多写少场景使用SQLite，如用户增长可迁移到PostgreSQL

### 集成点
- arXiv API: https://export.arxiv.org/api/query
- PubMed API: https://www.ncbi.nlm.nih.gov/books/NBK25501/
- Semantic Scholar API: https://api.semanticscholar.org/api-docs/
- Google OAuth: Google Identity Platform
- **AI服务：**
  - 智谱GLM API: https://open.bigmodel.cn/dev/api
  - OpenAI API: https://platform.openai.com/docs/api-reference
  - Ollama (本地): https://ollama.ai/

### 性能要求
- 搜索响应时间 < 2秒

### AI成本分析

#### GLM API 定价（智谱AI）

**官方定价：**
- 输入：¥0.5 / 100万 tokens
- 输出：¥0.5 / 100万 tokens

**实际使用成本计算：**

| 场景 | Token消耗 | 单次成本 | 说明 |
|------|----------|---------|------|
| 快速预览 | ~500 tokens | ¥0.00025 | 1-2句核心内容 |
| 简短摘要 | ~450 tokens | ¥0.00023 | 2-3句话，约100字 |
| 中等摘要 | ~1000 tokens | ¥0.0005 | 段落式，约300-500字 |
| 详细摘要 | ~2000 tokens | ¥0.001 | 全面总结，约800-1200字 |

**规模化成本预估：**

| 使用量 | 中等摘要成本 | 商业场景 |

#### 成本优化策略

**技术层面：**

2. **按需生成**：用户主动点击才生成，不自动生成

4. **混合模式**：热门文献用 Ollama（免费），冷门用 GLM

**产品层面：**
1. **免费配额**：每用户每天N次免费（如10次）
2. **会员制度**：
   - 免费用户：每天10次快速预览
   - 基础会员（¥9.9/月）：无限次 GLM 摘要
   - 高级会员（¥29.9/月）：GLM + 导出功能
3. **按需付费**：单次购买（如 ¥1.99 = 100次摘要）
